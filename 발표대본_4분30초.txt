=================================================
[A] 비전 + 문제정의  0:00 ~ 1:40 (100초)
=================================================
안녕하세요, "Tripia" 라는 ai여행 경로 추천 앱을 만든 임다솔입니다.

여러분, 주말 여행을 가려고 블로그 수십 개를 뒤지다가 새벽까지 일정을
못 정한 경험 있으신가요? 저는 있습니다. 그리고 이게 바로 Tripia를
만든 이유입니다.

Tripia의 비전은 "당신의 발걸음을 AI가 별자리처럼 이어, 이 세상
어디에도 없던 하나뿐인 여행을 그려드립니다"입니다.

일단 기존 여행 계획의 문제점은 세 가지입니다.

첫째, 계획을 짜느라 피로하다는 점 입니다. 여행지, 맛집, 동선을 따로따로 검색하다 보면
계획에만 몇시간이 걸립니다.

둘째, 획일적 추천입니다. 블로그나 유튜브는 사용자의 취향을 모르기 때문에
사람들이 많이 가는 동선을 획일적으로 추천합니다. 

셋째, 상황 대응이 어렵다는 점 입니다. 화창한 날을 기준으로 계획을 짯는데 
갑자기 비가 오면 처음부터 다시 짜야 하는 불편함이 있습니다. 

시간, 개인화, 실시간 대응 — 이 세 가지가 Tripia가 풀어야 할
문제였습니다.

=================================================
[B] 해결 방법 + 구현  1:40 ~ 3:20 (100초)
=================================================
Tripia는 이 세 가지 문제를 AI와 실시간 데이터로 해결합니다.

첫째, 취향 기반 개인화 추천입니다. 온보딩에서 고른 여행 스타일,
예산, 동행, 활동 강도, 실내·실외 선호를 바탕으로 홈 화면에 Gemini가
나라별 추천 카드를 만들어주고, 경로 검색 시에도 이 취향이 기본
필터로 자동 적용됩니다. 더 이상 '나와 안 맞는 추천'을 받지 않습니다.

둘째, AI 경로 생성입니다. 목적지와 기간만 입력하면 Gemini 2.5
Flash가 취향과 날씨를 반영해 10초 안에 동선을 짜줍니다. 응답에
마크다운이 섞여 와도 안전하게 파싱하는 _extractJson 함수를 만들어
안정성을 확보했습니다.

셋째, 실시간 날씨 반영입니다. 60개 이상 도시의 실시간 날씨를
받아오고, 갑자기 비가 오면 AI여행경로 화면에서 현재 날씨를 다시
확인한 뒤 실내·실외 필터를 바꿔 즉시 재추천을 받을 수 있습니다.

넷째, 지도 시각화입니다. flutter_map과 OpenStreetMap으로 API 키
없이 전체 동선을 한눈에 보여줍니다.

다섯째, 커뮤니티입니다. 경로 공유·질문·자유 게시판에 댓글과
좋아요·북마크 기능으로 다른 사용자의 여행 경로를 참고할 수 있습니다.

구조적으로는 Presentation, Data, Domain 3계층으로 설계했습니다.
예를 들어 "제주 2박3일"을 입력하면 RouteInputScreen이
GeminiService를 호출하고, 응답을 TripPlan으로 파싱해
RouteResultScreen에서 보여주고 저장합니다. 화면이 서비스를 직접
호출하는 단순하고 명확한 흐름입니다.

=================================================
[C] 기술·WBS·진행/ADR 요약  3:20 ~ 3:55 (35초)
=================================================
기술 스택은 Flutter Web, Dart, GoRouter, Gemini 2.5 Flash,
OpenWeatherMap, Unsplash, flutter_map, SharedPreferences입니다.

WBS상 인증, AI 경로 추천(개인화 포함), 실시간 상황 반영, 지도,
마이페이지, 커뮤니티, 문서·QA 7개 모듈을 9주차부터 15주차까지 모두
완성했습니다.

ADR은 총 4개입니다. Flutter 선택(ADR-0001)부터, 1인 개발과 7주
일정에 맞춰 Riverpod·Firebase 계획을 StatefulWidget·
SharedPreferences로 의도적으로 피벗한 ADR-0004까지 기록했습니다.

코드 품질은 flutter analyze 경고 0개, 단위·통합 테스트 38개 전부
통과합니다. Gemini 모델을 flash로 바꿔 응답 속도도 최적화했고,
로컬 실행은 클론 후 pub get, API 키 설정, flutter run 4단계로
바로 됩니다.
