# ADR-0001 — 모바일 프레임워크: Flutter

- **날짜**: 2026-05-12
- **상태**: 확정

---

## 배경

iOS와 Android를 동시에 지원하는 모바일 앱을 개발해야 한다.
지도, 실시간 위치, 카메라(커뮤니티 사진 업로드) 등 네이티브 기능이 필요하며,
커스텀 UI(경로 카드, 지도 오버레이)를 자유롭게 구성할 수 있어야 한다.

---

## 결정

**Flutter**를 선택한다.

---

## 대안

| 대안 | 검토 이유 | 제외 이유 |
|------|-----------|-----------|
| React Native | JS 생태계, 커뮤니티 풍부 | JS/TS 숙련도 없음, 네이티브 모듈 충돌 이슈 |
| 네이티브 (Swift + Kotlin) | 최고 성능, 플랫폼 완전 지원 | 두 코드베이스 유지 불가, 개발 기간 2배 |
| Kotlin Multiplatform | 비즈니스 로직 공유 | UI 레이어 공유 안 됨, 생태계 미성숙 |

---

## 결과

- **장점**: 단일 코드베이스로 iOS/Android 동시 빌드, Google Maps·Firebase Flutter SDK 공식 지원, Widget 기반 커스텀 UI 자유도 높음
- **단점**: Dart 언어를 새로 학습해야 함 → `05-risks.md` R-001 위험으로 등록
- **영향**: 상태관리 도구도 Flutter 생태계 안에서 선택해야 함 → ADR-0002 참조
